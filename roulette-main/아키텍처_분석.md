# Roulette-Main 아키텍처 분석

## 🎯 핵심 아키텍처

### 1. 진입점 (index.ts)
```
index.ts → Roulette 인스턴스 생성 → window.roulette에 노출
```

### 2. 메인 게임 클래스 (Roulette)
**역할**: 게임 전체를 관리하는 중앙 컨트롤러

**핵심 구조**:
- `EventTarget` 상속 → 커스텀 이벤트 발생 가능
- `requestAnimationFrame` 기반 게임 루프
- 고정 시간 간격 물리 시뮬레이션 (10ms)

**주요 컴포넌트**:
```typescript
- physics: IPhysics          // 물리 엔진 (Box2D-WASM)
- _renderer: RouletteRenderer // 렌더링 엔진
- _camera: Camera            // 카메라 시스템
- _marbles: Marble[]         // 구슬 배열
- _uiObjects: UIObject[]      // UI 컴포넌트들
- _particleManager            // 파티클 효과
```

**게임 루프 흐름**:
```
_update() {
  1. 시간 계산 (elapsed time)
  2. 물리 시뮬레이션 (physics.step)
  3. 구슬 업데이트 (_updateMarbles)
  4. 파티클 업데이트
  5. 효과 업데이트
  6. UI 업데이트
  7. 카메라 업데이트
  8. 렌더링 (_render)
  9. requestAnimationFrame 재귀 호출
}
```

### 3. 물리 엔진 (Box2dPhysics)
**역할**: Box2D-WASM을 사용한 물리 시뮬레이션

**핵심 메서드**:
- `init()`: Box2D 초기화
- `createStage()`: 맵 엔티티 생성 (정적/동적 바디)
- `createMarble()`: 구슬 생성 (동적 바디, 원형)
- `step()`: 물리 시뮬레이션 한 프레임 진행
- `impact()`: 스킬 효과 (폭발형 충격)

**엔티티 타입**:
- `static`: 정적 장애물 (벽, 바닥)
- `kinematic`: 움직이는 장애물
- `dynamic`: 구슬 (동적)

**Shape 타입**:
- `box`: 사각형
- `circle`: 원형
- `polyline`: 선분 연결 (경로)

### 4. 렌더링 시스템 (RouletteRenderer)
**역할**: Canvas 2D 렌더링

**렌더링 순서**:
```
1. 배경 채우기
2. 카메라 변환 적용
3. 엔티티 렌더링 (장애물)
4. 효과 렌더링
5. 구슬 렌더링
6. UI 오브젝트 렌더링
7. 파티클 렌더링
8. 승자 표시
```

**최적화**:
- 뷰포트 밖 구슬은 렌더링 스킵
- 이미지 캐싱 (스킨)
- 테마 기반 색상 관리

### 5. 구슬 시스템 (Marble)
**역할**: 개별 구슬의 상태 및 렌더링

**속성**:
- `name`: 이름
- `weight`: 가중치 (스킬 쿨타임에 영향)
- `hue`: 색상 (HSL)
- `skill`: 현재 스킬 상태
- `isActive`: 활성화 여부

**스킬 시스템**:
- `Impact`: 주변 구슬에 충격파
- 가중치가 높을수록 스킬 발동 확률 높음
- 쿨타임 시스템 (5초 기준)

**스틱 감지**:
- 위치 변화가 거의 없으면 스틱으로 판단
- 5초 이상 스틱이면 자동으로 흔들기

### 6. 카메라 시스템 (Camera)
**역할**: 뷰포트 관리 및 구슬 추적

**기능**:
- 구슬 자동 추적
- 목표 지점 근처에서 줌인
- 부드러운 보간 (interpolation)
- 미니맵 연동

**줌 로직**:
- 목표 구슬이 결승선 근처에 오면 자동 줌인
- 거리에 따라 줌 레벨 조정

### 7. 맵 시스템 (maps.ts)
**구조**:
```typescript
StageDef {
  title: string
  entities: MapEntity[]  // 장애물들
  goalY: number          // 결승선 Y 좌표
  zoomY: number          // 줌인 시작 Y 좌표
}
```

**엔티티 정의**:
- 위치 (position)
- 모양 (shape: box/circle/polyline)
- 물리 속성 (density, restitution, angularVelocity)
- 생명력 (life: -1이면 영구, >0이면 터치 시 제거)

### 8. UI 시스템 (UIObject)
**인터페이스**: 모든 UI 컴포넌트의 기본

**구현체**:
- `RankRenderer`: 순위 표시 (스크롤 가능)
- `Minimap`: 미니맵 (드래그로 뷰포트 이동)
- `FastForwader`: 빠르게 감기 (중앙 클릭/터치)

**이벤트 처리**:
- 마우스 이벤트 (Move, Down, Up, DblClick)
- 휠 이벤트
- 바운딩 박스 기반 충돌 감지

### 9. 게임 플로우

**초기화**:
```
1. Roulette 생성자
2. Renderer 초기화 (이미지 로드)
3. Physics 초기화 (Box2D 로드)
4. 맵 로드
5. UI 오브젝트 등록
6. 이벤트 리스너 연결
7. 게임 루프 시작
```

**게임 시작**:
```
1. setMarbles() - 구슬 생성
   - 이름 파싱 (가중치, 개수)
   - 가중치 정규화
   - 순서 셔플
   - 물리 바디 생성 (비활성 상태)
   
2. start() - 게임 시작
   - 물리 엔진 활성화
   - 구슬 활성화
   - 카메라 추적 시작
   - 녹화 시작 (옵션)
```

**게임 진행**:
```
매 프레임:
1. 물리 시뮬레이션 (10ms 간격)
2. 구슬 위치 업데이트
3. 결승선 도착 체크
4. 승자 결정 (winnerRank)
5. 시간 스케일 조정 (슬로우 모션)
6. 카메라 업데이트
7. 렌더링
```

**승자 결정**:
- `winnerRank`: 목표 순위 (0 = 첫 번째)
- 결승선 도착 시 `_winners` 배열에 추가
- 목표 순위 도달 시 게임 종료

### 10. 핵심 유틸리티

**parseName()**: 이름 파싱
- `이름/가중치*개수` 형식
- 예: `짱구/5*3` → name="짱구", weight=5, count=3

**shuffle()**: 배열 셔플 (Fisher-Yates)

**시간 관리**:
- 고정 시간 간격 (10ms)
- 경과 시간 누적
- 시간 스케일 (슬로우 모션)

## 🔄 Race 게임에 도입 시 고려사항

### 공통점
- 물리 엔진 (Box2D)
- 구슬 시스템
- 렌더링 파이프라인
- 카메라 시스템

### 차이점
- **Roulette**: 수직 낙하, 결승선 도착
- **Race**: 수평 경주, 결승선 도착

### 필요한 수정사항
1. **맵 구조**: 수평 경주 트랙
2. **카메라**: 수평 추적
3. **구슬 배치**: 시작선에 일렬 배치
4. **승자 로직**: 동일 (첫 번째 도착자)

### 재사용 가능한 컴포넌트
- ✅ `Box2dPhysics` (물리 엔진)
- ✅ `Marble` (구슬 클래스)
- ✅ `RouletteRenderer` (렌더러 - 약간 수정)
- ✅ `Camera` (카메라 - 수평 추적으로 수정)
- ✅ `UIObject` 시스템
- ✅ 맵 시스템 (구조는 동일, 데이터만 변경)
