# Marble Roulette 실행 가이드

## 프로젝트 개요

**Marble Roulette**은 구슬을 떨어뜨려 추첨하는 웹 기반 게임입니다. TypeScript로 작성되었으며, Box2D 물리 엔진을 사용하여 실시간 구슬 시뮬레이션을 구현합니다.

- **프로젝트 타입**: TypeScript 웹 애플리케이션
- **빌드 도구**: Parcel 2.16.1
- **물리 엔진**: box2d-wasm 7.0.0
- **패키지 매니저**: yarn (권장) 또는 npm
- **개발 서버 포트**: 1235

---

## 사전 요구사항

1. **Node.js** (v24.11.1 이상 권장)
2. **yarn** 또는 **npm** 패키지 매니저

### yarn 설치 방법 (권장)

```bash
# npm을 통해 yarn 설치
npm install -g yarn

# 설치 확인
yarn --version
```

## 실행 방법

### 1단계: 프로젝트 디렉토리로 이동

```bash
cd /Users/simsimi/git/pinball/roulette-main
```

### 2단계: 의존성 설치

**yarn 사용 (권장):**
```bash
yarn install
```

**npm 사용:**
```bash
npm install
```

이 명령어는 다음 의존성들을 설치합니다:
- **개발 의존성**: Parcel, TypeScript, ESLint, Prettier 등
- **런타임 의존성**: box2d-wasm

### 3단계: 개발 서버 실행

**yarn 사용:**
```bash
yarn dev
```

**npm 사용:**
```bash
npm run dev
```

개발 서버가 **포트 1235**에서 시작됩니다.

### 4단계: 브라우저에서 접속

브라우저에서 다음 주소로 접속하세요:
```
http://localhost:1235
```

## 사용 가능한 스크립트

### 개발 모드 (`yarn dev` 또는 `npm run dev`)
- 개발 서버를 포트 1235에서 실행
- Hot Module Replacement (HMR) 지원
- 실시간 코드 변경 반영
- 소스맵 생성

### 프로덕션 빌드 (`yarn build` 또는 `npm run build`)
```bash
yarn build
```

빌드 과정:
1. Parcel로 TypeScript를 JavaScript로 컴파일
2. SCSS를 CSS로 변환
3. `/roulette/` 경로를 기준으로 빌드
4. Service Worker 빌드 (`scripts/build-sw.js` 실행)
5. 결과물은 `dist/` 폴더에 생성

### 린트 검사 (`yarn lint` 또는 `npm run lint`)
```bash
yarn lint
```
- TypeScript 코드의 린트 검사 및 자동 수정
- ESLint와 Prettier 규칙 적용

## 프로젝트 구조

```
roulette-main/
├── src/                    # 소스 코드
│   ├── index.ts           # 진입점
│   ├── roulette.ts        # 룰렛 게임 로직
│   ├── physics-box2d.ts   # 물리 엔진 통합
│   ├── data/              # 게임 데이터
│   │   ├── constants.ts   # 상수 정의
│   │   ├── languages.ts   # 다국어 지원
│   │   └── maps.ts        # 맵 데이터
│   ├── types/             # TypeScript 타입 정의
│   └── utils/             # 유틸리티 함수
├── assets/                # 정적 파일
│   ├── images/            # 이미지 리소스
│   └── style.scss         # 스타일시트
├── scripts/               # 빌드 스크립트
│   └── build-sw.js        # Service Worker 빌드
├── index.html             # HTML 엔트리 포인트
├── package.json           # 프로젝트 설정 및 의존성
├── tsconfig.json          # TypeScript 설정
├── .parcelrc              # Parcel 설정
└── dist/                  # 빌드 결과물 (자동 생성)
```

## 주요 기능

1. **구슬 추첨 시스템**
   - 이름 입력 및 가중치 설정 (`/숫자` 형식)
   - 개수 설정 (`*숫자` 형식)
   - 예: `짱구*5, 짱아*10, 봉미선*3`

2. **게임 옵션**
   - 첫 번째/마지막/커스텀 순위 설정
   - 스킬 사용 옵션
   - 다크모드
   - 자동 녹화
   - 맵 선택

3. **물리 시뮬레이션**
   - Box2D-WASM 물리 엔진
   - 실시간 구슬 충돌 및 중력 시뮬레이션

4. **PWA 지원**
   - Service Worker를 통한 오프라인 지원
   - 모바일 앱처럼 설치 가능

## 문제 해결

### 포트 1235가 이미 사용 중인 경우

다른 포트로 실행하려면:
```bash
parcel index.html --port 3000
```

### 의존성 설치 오류

캐시를 지우고 다시 시도:
```bash
# yarn 사용
yarn cache clean
yarn install

# npm 사용
npm cache clean --force
npm install
```

### TypeScript 컴파일 오류

타입 체크만 수행:
```bash
npx tsc --noEmit
```

### 빌드 오류

`.parcel-cache` 폴더 삭제 후 재시도:
```bash
rm -rf .parcel-cache
yarn build
```

## 개발 팁

1. **실시간 개발**
   - 코드를 수정하면 브라우저가 자동으로 새로고침됩니다
   - 콘솔에서 실시간 오류를 확인할 수 있습니다

2. **디버깅**
   - 브라우저 개발자 도구의 Sources 탭에서 TypeScript 소스를 확인할 수 있습니다
   - `window.roulette` 객체를 통해 게임 인스턴스에 접근 가능

3. **로컬 스토리지**
   - 입력한 이름은 자동으로 로컬 스토리지에 저장됩니다
   - `localStorage.getItem('mbr_names')`로 확인 가능

## 배포

빌드 완료 후 `dist/` 폴더의 내용을 웹 서버에 업로드하면 됩니다.

빌드 결과물은 `/roulette/` 경로를 기준으로 생성되므로, 서버 설정 시 주의가 필요합니다.
